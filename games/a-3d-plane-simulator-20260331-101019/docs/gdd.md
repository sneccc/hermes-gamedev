# Game Design Document: Vector Wings (3D Plane Simulator)

## Overview
**Title:** Vector Wings
**Genre:** 3D Flight Simulator / Arcade Action
**Engine:** Python + Raylib (`pyray`)
**Scope:** 4-8 Python files, purely procedural graphics (no external assets)

## Core Gameplay
The player controls a simple 3D aircraft composed of geometric primitives. The objective is to fly through a sequence of floating rings within a time limit. Every ring passed increases the player's score and adds bonus time to the clock.

### Objectives
- **Win Condition:** Accumulate 20 rings to reach the score goal, or play endlessly until time runs out (high score mode).
- **Lose Condition:** Run out of time, or crash into the ground.

### Controls
- **W / S:** Pitch up / Pitch down
- **A / D:** Roll left / Roll right
- **Q / E:** Yaw left / Yaw right
- **Space / Shift:** Accelerate (Throttle up)
- **Ctrl / Alt:** Decelerate (Throttle down)

## Mechanics
- **Flight Model:** A simplified arcade physics model. The plane has a forward velocity based on throttle. Pitch, roll, and yaw apply rotational velocity. Gravity pulls the plane downward slowly, requiring lift (generated by forward speed) to stay aloft.
- **Rings (Waypoints):** Generated dynamically in front of the player's flight path. Only one or two rings are visible/active at a time.
- **Time/Score:** Start with 30 seconds. Passing a ring grants +1 point and +5 seconds.
- **Collisions:** A bounding sphere around the plane checks collision against the ground plane (y <= 0) and the bounding boxes of the rings.

## Aesthetics and Audio
- **Visuals:** Low-poly aesthetic using built-in raylib 3D drawing functions (`DrawCube`, `DrawCylinder`, `DrawGrid`).
- **Plane:** A combination of a cylinder (fuselage) and flat boxes (wings, tail).
- **Environment:** A large flat ground represented by `DrawGrid` or a green plane.
- **Feedback:** Screen flash or text pop-up when passing a ring. "GAME OVER" or "YOU WIN" text overlay.
- **Audio:** No external audio required. (Can use procedural beeps if necessary, but visual feedback is primary).

## File Structure & Scope
- `main.py`: Entry point and main game loop.
- `constants.py`: Constants for window size, colors, tuning.
- `entities.py`: Classes for the Plane and Rings.
- `systems.py`: Spawning, collision detection, game state management.
- `utils.py`: Vector math, rotation matrices (quaternions if needed for 6DOF flight).
- `ui.py`: 2D HUD (score, time, speed, altitude, crosshair).
